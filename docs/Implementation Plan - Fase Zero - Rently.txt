# Implementation Plan

## P00 — Fundação do Repositório
- P00.01 — Inicialização do Git — Criar repositório raiz do projeto com controle de versão Git.
- P00.02 — Estrutura de Pastas Raiz — Criar diretórios /backend, /frontend, /docs e /docker.
- P00.03 — Gitignore Combinado — Criar .gitignore cobrindo artefatos de build do .NET e do Angular.
- P00.04 — README Inicial — Criar README.md descrevendo o projeto, a stack e o setup local.

## P01 — Backend: Estrutura da Solution
- P01.01 — Solution .NET — Criar Rently.sln na pasta /backend.
- P01.02 — Projeto Rently.Domain — Criar projeto de domínio com subpastas por módulo (Imoveis, Contratos, Locatarios, Usuarios).
- P01.03 — Projeto Rently.Application — Criar projeto de aplicação com subpastas Commands e Queries por módulo.
- P01.04 — Projeto Rently.Infrastructure — Criar projeto de infraestrutura com subpastas Persistence e Security.
- P01.05 — Projeto Rently.CrossCutting — Criar projeto compartilhado para exceções, constantes e extensões.
- P01.06 — Projeto Rently.Api — Criar projeto Web API de entrada da aplicação.
- P01.07 — Referências entre Projetos — Configurar referências: Api → Application → Domain; Infrastructure → Application e Domain; CrossCutting referenciado por todos.

## P02 — Backend: Convenções e Configuração
- P02.01 — Directory.Build.props — Definir versão do C#, nullable enable e warnings como erro.
- P02.02 — Arquivos de Configuração — Criar appsettings.json, appsettings.Development.json e appsettings.Production.json.
- P02.03 — Connection String — Configurar connection string do SQL Server via appsettings e variáveis de ambiente.
- P02.04 — EF Core Inicial — Configurar EF Core code-first e criar a migration InitialCreate vazia.
- P02.05 — Autenticação JWT — Configurar middleware de autenticação/autorização JWT exigindo token válido por padrão.
- P02.06 — Dockerfile Backend — Criar Dockerfile multi-stage para Rently.Api.
- P02.07 — Estrutura para Testes Futuros — Preparar pastas compatíveis com futura inclusão de Rently.UnitTests e Rently.IntegrationTests.

## P03 — Frontend: Workspace Angular
- P03.01 — Workspace Angular — Criar workspace Angular 22 com o projeto "rently" em /frontend.
- P03.02 — Pasta Core — Criar estrutura /core para serviços singleton, interceptors, guards e modelos globais.
- P03.03 — Pasta Shared — Criar estrutura /shared para componentes, diretivas e pipes reutilizáveis.
- P03.04 — Pasta Features — Criar estrutura /features com subdiretórios por módulo funcional de negócio.
- P03.05 — Pasta Layout — Criar estrutura /layout para componentes estruturais (header, sidebar, footer).

## P04 — Frontend: Configurações Iniciais
- P04.01 — Tema e Paleta de Cores — Definir variáveis SCSS globais em tons de azul petróleo.
- P04.02 — Roteamento Base — Configurar rotas públicas e privadas com guard de autenticação.
- P04.03 — Interceptor HTTP JWT — Configurar interceptor para anexar token JWT e tratar respostas 401/403.
- P04.04 — Arquivos de Ambiente — Configurar environment.ts e environment.prod.ts com a URL base da API.
- P04.05 — Uso Pontual de RxJS — Aplicar RxJS apenas em streams de estado e debounce de buscas.
- P04.06 — Dockerfile Frontend — Criar Dockerfile de build de produção servido via Nginx.
- P04.07 — Estrutura para Testes Futuros — Preparar estrutura compatível com futura inclusão de testes unitários e e2e.

## P05 — Ambiente Docker
- P05.01 — Docker Compose — Criar docker-compose.yml orquestrando backend, frontend e SQL Server.
- P05.02 — Variáveis e Volumes — Definir variáveis de ambiente e volumes para persistência e hot-reload.
- P05.03 — Validação do Ambiente — Validar subida completa via docker-compose up com comunicação entre serviços.

## P06 — Preparação para Nuvem
- P06.01 — Documento de Visão Kubernetes — Criar documento em /docs registrando premissas de implantação futura (escalabilidade horizontal, separação de imagens por serviço, secrets/configmaps).
